## 安全至上

大多数开源导航站项目在安全性上投入不足。我们采取了不同的策略。

#### 传统问题

- 使用简单的密码验证
- 容易受到 SQL 注入攻击
- 容易受到 XSS 攻击
- 没有速率限制
- 输入验证不完整

#### NavTools 的安全措施

经过 **14 个安全修复提交**，包括：

##### 认证与授权
- **JWT 签名**：使用强密钥签名的 JSON Web Tokens
- **bcrypt 哈希**：密码存储使用工业级哈希算法
- **HttpOnly Cookie**：防止 XSS 窃取 Token

##### 攻击防护
- **SQL 注入防护**：D1 绑定参数，杜绝注入
- **XSS 防护**：React 自动转义 + 严格内容安全策略
- **CSRF 防护**：SameSite Cookie 策略
- **SSRF 防护**：严格验证 URL 格式

##### 访问控制
- **速率限制**：防止暴力破解（5次/15分钟）
- **访客模式**：数据最小化暴露

## 适合谁？

### 不适合
❌ 需要服务器端渲染 (SSR) 做 SEO 的大型门户（NavTools 是 SPA）
❌ 需要存储大量文件/图片（NavTools 仅存储链接和 URL）
❌ 不想依赖 Cloudflare 的人

### 完美适合
✅ **开发者**：喜欢折腾，喜欢新技术
✅ **学生/个人**：预算有限，追求零成本
✅ **团队/小公司**：需要内部导航，注重安全和权限
✅ **效率控**：追求极致的加载速度
